Global objects

  

Here you can adjust settings related to the use of E-commerce module global objects.

 

You can use both site-specific objects available for your individual sites only and global objects shared across all these sites. From the point of view of the E-commerce module site separation, there are three groups of objects available for a particular website:

 

Objects with both site and global option

The group includes E-commerce module objects which can be used either as site-specific or site-specific together with global; corresponds to the Allow global objects for section on this page.

Objects with only site or global option

The group includes E-commerce module objects which can be used only separately, either as site-specific or global; corresponds to the Use global settings for section on this page.

Objects with special cases

The group includes E-commerce module objects which cannot be put into either of the above groups; e.g. customers, orders, reports.

 

The following settings can be adjusted:

 

Allow global objects for

Products

Indicates if global products are allowed.

Product options

Indicates if global product options are allowed.

Manufacturers

Indicates if global manufacturers are allowed.

Suppliers

Indicates if global suppliers are allowed.

Discount coupons

Indicates if global discount coupons are allowed.

Discount levels

Indicates if global discount levels are allowed.

Departments

Indicates if global departments are allowed.

Shipping options

Indicates if global shipping options are allowed.

Payment methods

Indicates if global payment methods are allowed.

Use global settings for

Tax classes

Indicates if global tax classes are used.

Currencies

Indicates if global currencies are used.

Exchange rates

Indicates if global exchange rates are used.

Customers credit

Indicates if customer can use global credit.

Order statuses

Indicates if global order statuses are used.

Public statuses

Indicates if global public statuses are used.

Internal statuses

Indicates if global internal statuses are used.

Invoice

Indicates if global invoices are used.
